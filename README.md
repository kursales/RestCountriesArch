1. Несколько лет в андроид разработке, участвовал в больших коммерческих проектах, вернулся после перерыва
2. Использовал разные команды, часть из них add, revert, clone, diff, fetch, push, reset и так далее
3. Коллекции, оба реализуют логику словарей храня данные в виде ключ значения, HashSet та же hashMap,
   но ключом также выступает значение, HashMap под капотом устоена в виде бакетов, в которые будут складываться разные по иквалс, но одинаковые по хеш коду элементы.
4. Легковесные потоки, могут выполняться на одном потоке, но чаще используют пул потоков, быстрота обеспечивается прерываемостью, корутина сохраняется в объект Continuation,
   при завершении ожидания, диспатчер возвращает его корутине и работа продолжается с того же места, пока корутина ждет, поток освобожается для других задач,
   логика переключения состояний реализована как простая стейт машина. Состоят из Диспатчера, джобы и контекста, запускаются в скоупе
